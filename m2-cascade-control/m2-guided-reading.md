## 0. 先抓主線：一份文件，三條暗線

```
結構：四層級聯誰是誰的 setpoint    → §0/§1/§2 方塊圖
分工：哪兩層只有 P、哪兩層完整 PID  → §0 直覺 + §3
查修：看症狀 → 指到環與增益 → 怎麼調 → §4 對照表 + §5 順序
        看懂結構 ──► 記住分工 ──► 會查症狀
```

**一句話**：M2 只放大 [M1 dataflow](m1-dataflow-diagram.md) §1「級聯控制器」那一段——M1 看「命令走哪條路」，**M2 看「每一環在算什麼、調哪個參數」**。先把 §0 的一句話總覺記死，整份就有骨架。

**核心直覺（先背這句）**：**外兩層（位置/姿態）只有 P，內兩層（速度/角速度）才用完整 PID；最內的角速度環是飛穩的命脈，調參從它開始。**

**建議順序與預算（純讀約 45–60 分）**：§0+§1/§2 方塊圖(20 分) → §3 破混淆(10 分) → §4 對照表當工具查(15 分) → 本檔 §3 小考(15 分)。

## 1. 帶讀：[cascade-control-diagram](m2-cascade-control-diagram.md)

**怎麼讀**：方塊圖（§1 Mermaid / §2 ASCII）只是把 §0 那句話畫出來——別急著背參數名，先把「四層、誰餵誰、哪層 P 哪層 PID」這個形狀刻進腦。§4 對照表不是用背的，是**當字典查**：看到症狀來翻。

**讀到這裡停一下 ⏸**
- 看完 §0/§1 → 闔上，**默畫四層方塊**：標出每層回饋來源（誰相減成誤差）、輸出什麼 setpoint、哪層只有 P。
- 看完 §3 → 問：**`MC_ROLL_P` 和 `MC_ROLLRATE_P` 差在哪？**（前者＝姿態環/控**角度**/只 P；後者＝角速度環/控**角速度→力矩**/完整 PID。記法：有 `RATE` 的才是最內力矩級。）
- 看完 §4a → 問：**機身高頻抖動、嗡嗡聲、馬達發燙，先懷疑誰？**（內環角速度的 `P`（太大）或 `D`（放大噪聲）。）
- 看完 §5 → 問：**調參為何「由內到外」？**（內環不穩，外層怎麼調都白搭。）

**白話比喻（卡住時想這個）**
- 級聯＝**套娃/接力**：外層喊「要去哪」（慢、只用 P 給方向），把目標交給內層；內層喊「穩不穩」（快、用 PID 出力）。最內角速度環是**命脈**。
- **P 找速度、I 找穩態、D 找阻尼**：P 太猛交給 D 收，長期歪了交給 I 補。

**最容易誤會**
- ❌ 以為四層都要 PID → ✅ **外兩層（位置/姿態）只有 P**。
- ❌ 以為「調 roll」只有一個參數 → ✅ 有**兩個不同樓層的 roll P**（`MC_ROLL_P` vs `MC_ROLLRATE_P`），先把內環調穩外環才有意義。
- ❌ 從外環（位置）開始調 → ✅ **由內到外**：④角速度 → ③姿態 → ②速度 → ①位置。
- ❌ 以為角速度回饋走 EKF2 → ✅ **gyro 濾波直連**（呼應 [M1](m1-dataflow-diagram.md) §2「最內環要最低延遲」）。

**讀完該能答**：不看表，講出四層各自「P only 還是 PID」，並把「高頻抖 / 遲鈍軟趴 / 撐不住慢慢歪」各指到正確的環與增益。

## 2. 把 M2 串進整條線（讀完務必合眼想一遍）

```
M1 dataflow §1「級聯控制器」這一小段
        │  放大成整份 ▼
M2：位置(P)→速度(PID)→姿態(P)→角速度(PID)→分配→馬達
        ▲ 命令來源              ▲ 動作來源
        │ 由 M1 的 setpoint 灌入   │ M3 的 RL(Swift) 輸出 CTBR=推力+角速度
        │                         │   正好打進「角速度環(內環)」
   M1（手：送命令）          M3（眼睛/反射：學出動作）
```

- **往上游接 M1**：M2 的最外層「位置目標」就是 M1 送的 `TrajectorySetpoint`；M2 是 M1 dataflow 的**特寫鏡頭**。
- **往前沿接 M3**：[M3 五點筆記](m3-ai-models-comparison.md) 裡 **Swift 的 RL 輸出是 CTBR（集體推力 + body rate）**——那不是憑空，而是**直接餵進 M2 的最內角速度環**。讀懂 M2 內環，才懂為何 RL 政策能接上飛控。

> 記法：**外環決定「要去哪」、內環決定「穩不穩」；振動先看內環、遲鈍先看 P、撐不住先看 I。**

## 3. 分級小考（闔上文件再做；答不出就回對應段）

**🟢 Level 1 · 結構與分工**
1. 四層由外到內是哪四個？哪兩層只有 P、哪兩層完整 PID？
2. 為什麼角速度環最關鍵、最先調？

**🟡 Level 2 · 破混淆**
3. 一句話分清 `MC_ROLL_P` 與 `MC_ROLLRATE_P`（所屬環/控什麼/P 還是 PID）。
4. 「P 找＿、I 找＿、D 找＿」填空。

**🟡 Level 3 · 查症狀**
5. 高頻抖動 + 馬達發燙 → 哪環哪增益、偏大還偏小？
6. 逆風會慢慢歪、穩態撐不住 → 哪個增益不足？
7. 到 waypoint 過衝、繞著點振盪 → 哪環哪增益？

**🔴 Level 4 · 串接**
8. 調參的正確順序與心法是什麼？為何由內到外？
9. M2 的最外層目標由 [M1](m1-dataflow-diagram.md) 的哪個 message 提供？
10. [M3 的 Swift](m3-ai-models-comparison.md) RL 輸出的 CTBR（推力+角速度）會打進 M2 的哪一環？這說明了什麼？

> 對照答案：Q1–2 → §0/§1；Q3–4 → §3/§5；Q5–7 → §4a/§4c/§4d；Q8 → §5；Q9–10 → 本檔 §2 + [M1](m1-dataflow-diagram.md)/[M3](m3-ai-models-comparison.md)。
> **能答 1–7 → Module 2 因果理解到位；能答 8–10 → 看見 M2 上下游接口，可進 [Module 3 / Phase 3 AI 感知](m3-ai-models-comparison.md)。**
